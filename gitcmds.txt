Step 1: The "Where am I?" (Check your base)
Before you start a new feature, always make sure your main branch is up to date with the server.
git checkout main
git pull origin main
Step 2: The "Split" (Create a feature branch)
Now, create a new branch. Let's say you're adding a reporting feature using a Postgres Window Function.
git checkout -b feature/reporting-window-func
Step 3: The "Work" (Commit as usual)
Now you are in your safe "timeline." Edit your .cs files, write your raw lowercase SQL, and commit.
